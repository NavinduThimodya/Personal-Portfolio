1. Project Overview

This project encompassed the physical relocation of mission-critical Oracle Exadata infrastructure and the enhancement of its support capabilities. The scope included the safe migration of Exadata nodes to the Pitipana Data Center, ensuring business continuity. Subsequently, the project focused on optimizing system maintenance through the configuration of Auto Service Requests (ASR) and the execution of essential patch upgrades to maintain security and performance standards.

2. Technical Scope

Data Center Migration: Relocation of high-performance Exadata Engineered Systems between data centers, involving complex power and network reconfiguration.
Support Automation: Implementation of Oracle Auto Service Request (ASR) to automate the generation of service requests for hardware faults, significantly reducing potential downtime.
Lifecycle Management: Execution of Exadata patch upgrades to ensure software currency and stability.

3. My Contributions

Phase 1: Exadata Physical Migration

  • Decommissioning: Assisted in the shutdown procedures, powering down the Exadata nodes and cells at the source location.
  • Relocation & Re-deployment: Reattached all network cables and power connections at the Pitipana Data Center.
  • System Initialization: Performed the power-up sequence, ensuring all components came online correctly.

Phase 2: Patch Management

  • Patch Execution: Carried out an Exadata patch upgrade at the client site.
  • Stakeholder Coordination: Acted as the intermediary between Oracle Support and the Amana Bank team to ensure successful execution.

Phase 3: ASR Configuration

  • Configuration: Configured the Auto Service Request (ASR) feature within the My Oracle Support interface.
  • Implementation: Installed and registered the ASR manager, prepared configuration files, and verified setup through test events.
  • Optimization: Established a proactive support mechanism for automatic hardware fault service requests.
