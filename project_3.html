1. Project Overview

This project involved a critical infrastructure update for the client's backup environment. It consisted of two major phases: first, migrating the existing NetBackup Master Server from an old server to a new server running Oracle Linux, and second, upgrading the NetBackup software to a newer version. The project required deep troubleshooting of file system architectures and software version compatibility to ensure data integrity and system availability.

2. Technical Scope

Infrastructure Migration: Relocating the NetBackup Master Server to new hardware/OS infrastructure using Disaster Recovery (DR) methodologies.
Software Upgrade: Upgrading the NetBackup Master Server software while ensuring compatibility with existing client servers.
Disaster Recovery Simulation: Creating isolated test environments to simulate failure scenarios and validate migration procedures before production implementation.

3. My Contributions

Phase 1: NetBackup Migration

  • Problem Diagnosis: Identified file system mismatch (ext4 vs xfs) as the root cause of initial migration failures by analyzing installation logs.
  • Simulation & Testing: Created a simulation environment to replicate the issue and validated a solution by ensuring consistent file systems (ext4) across source, destination, and NFS shares.
  • Execution: Successfully executed the migration using the validated method.

Phase 2: Master Server Upgrade

  • Upgrade Execution: Initiated the upgrade of the CDB Master Server.
  • Troubleshooting: Diagnosed a version compatibility issue caused by an outdated plugin that halted the upgrade.
  • Resolution: Coordinated with Veritas support to identify and update the incompatible plugin, allowing the upgrade to complete successfully.

4. Technologies & Tools Used

Veritas NetBackup Oracle Linux Disaster Recovery (DR) NFS File Systems (ext4/xfs)
