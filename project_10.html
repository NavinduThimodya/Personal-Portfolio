“Thilinapath” Multi Campaign App
Deployment (On-Prem DevOps)

Project Details

Client: Ministry of Education, Sri Lanka
Location: On-Premises Data Center (Ministry Infrastructure)
Period: June 2025

1. Project Overview

This project involved the architectural design and initial deployment of “Thilinapath,” a multi-campaign application serving the Ministry of Education. The core objective was to establish a robust, on-premises DevOps environment to support the application's full lifecycle across Development (Dev), User Acceptance Testing (UAT), and Production (Prod) environments. The initiative focused on modernizing the deployment workflow using container orchestration and automated CI/CD pipelines to ensure high availability and rapid delivery.

2. Technical Scope

Container Orchestration: Implementation of RKE2 (Rancher Kubernetes Engine 2) for secure and compliant Kubernetes clustering, utilizing MetalLB for bare-metal load balancing.
High Availability & Data: Deployment of Redis clusters for caching and PCS clusters (Pacemaker/Corosync) for high-availability resource management, backed by NFS file servers for shared storage.
DevOps Toolchain: Establishment of a complete CI/CD ecosystem including a Private Docker Registry, Jenkins for continuous integration, and ArgoCD for GitOps-based continuous delivery.
Observability: Implementation of the ELK Stack (Elasticsearch, Logstash, Kibana) for centralized logging and monitoring.

3. My Contributions

Phase 1: Infrastructure Provisioning & Virtualization

  • Environment Setup: I provisioned and configured Ubuntu Virtual Machines (VMs) to serve as the host nodes for the cluster infrastructure.
  • Storage Configuration: I deployed NFS file servers to provide persistent, shared storage volumes required by the Kubernetes pods and application data.

Phase 2: Cluster Initialization & High Availability

  • Kubernetes Deployment: I set up the RKE2 clusters, ensuring a hardened and secure Kubernetes environment suitable for government deployment standards.
  • Load Balancing: I configured MetalLB to provide external IP addresses and load balancing services within the on-premises network.
  • Clustering: I deployed Redis clusters to handle application caching needs and set up PCS clusters to ensure high availability and failover capabilities for critical services.

Phase 3: DevOps Toolchain & Pipeline Configuration

  • CI/CD Implementation: I established the initial CI/CD pipelines using Jenkins for build automation and ArgoCD to manage deployments across Dev, UAT, and Prod environments using GitOps principles.
  • Artifact Management: I set up a Private Docker Registry to securely store and manage the application's container images.
  • Observability Setup: I deployed the ELK Stack, configuring the initial log aggregation pipelines to ensure visibility into system performance and application logs from day one.

4. Technologies & Tools Used

RKE2 Kubernetes MetalLB Redis PCS Clusters NFS Jenkins ArgoCD Docker Registry ELK Stack Ubuntu
